Frequently Asked Questions

1.The difference between var, let, and const
var: Function-scoped, can be redeclared and updated, hoisted with undefined.
let: Block-scoped, can be updated but not redeclared, hoisted without initialization.
const: Block-scoped, cannot be updated or redeclared, must be initialized during declaration.
2.The difference between map(), forEach(), and filter()
map(): Returns a new array by transforming each element using a callback function.
forEach(): Iterates over an array without returning a new array, just executes a function on each item.
filter(): Returns a new array containing only elements that satisfy a given condition.
3.Explain arrow functions and how they are different from regular functions
Arrow functions are a concise way to write functions using => syntax. They don't have their own this, so they inherit this from their surrounding scope, unlike regular functions. Additionally, arrow functions cannot be used as constructors and do not have the arguments object.
4.How JavaScript Promises work
JavaScript Promises represent a value that may not be available yet but will be resolved in the future. A promise has three states: pending, resolved (fulfilled), or rejected. You can handle the result using .then() for success or .catch() for errors once the promise is settled.
5.How closures work in JavaScript
Closures in JavaScript allow a function to remember and access variables from its lexical scope even after the outer function has finished execution. This means inner functions can access variables from their outer function even when called outside of it. Closures are often used for data encapsulation and maintaining state.
